# 👤 员工画像自动生成指南

本文档说明如何基于访谈内容自动生成**普通员工画像**。

> **v2.7.0 说明**：从 v2.6.0 版本开始，画像系统采用双轨制：
> - **管理干部（is_manager=True）** → 使用 [管理干部画像指南](manager-profile-guide.md)
> - **普通员工（is_manager=False）** → 使用本文档
>
> Step 1 会自动识别被访谈者是否为管理干部，并选择对应的画像模板。
>
> **v2.7.0 增强**：新增关键指标概览（工作投入度、异动风险、战场匹配度），新增基础身份信息字段（管理主体、招聘类型），与管理干部画像结构对齐。

## 📋 画像结构

### 1. 基础画像信息

| 维度 | 数据来源 | 示例 |
|------|---------|------|

| **招聘类型** | 访谈记录/元数据（如可提取） | 校招/社招 |

### 1.5 关键指标概览（v2.7.0 新增）

> **说明**：与管理干部画像对齐的核心指标快速概览表，帮助 HRBP 一目了然地掌握员工核心状态。

| 维度 | 数据来源 | 评估方法 | 输出格式 |
|------|---------|---------|---------|
| **工作投入度** | 时间投入、精力投入、情感投入的相关描述 | 综合分析 | [高/中/低] + 描述 |
| **异动风险** | 不满情绪、外部机会提及、职业发展受阻等信号 | 信号强度综合评估 | [高/中/低] + 描述 |
| **战场** | 工作领域、核心职责描述 | 直接提取 | 文字描述 |
| **战场匹配度** | 能力特长与岗位要求的对比分析 | 对比分析 | [高/中/低] + 描述 |
| **个人优劣势摘要** | 综合能力层面的优势与不足 | 事实提取 | 关键词概括 |
| **发展诉求摘要** | 员工表达或推断的发展期望 | 直接提取+推断 | 一句话概括 |

**工作投入度评估维度：**
- 时间投入（权重30%）：加班/工作时长描述
- 精力投入（权重35%）：主动性/深入度/亲力亲为程度
- 情感投入（权重35%）：热情/成就感/使命感/倦怠感

**异动风险信号分类：**
| 信号类型 | 高风险信号 | 中风险信号 | 低风险信号 |
|---------|-----------|-----------|-----------|
| 不满情绪 | "想换个环境"、"看不到希望" | "有点疲惫"、"不太满意" | "虽然辛苦但值得" |
| 外部机会 | "有人挖我"、"在看机会" | "朋友推荐过"、"不排斥好机会" | "没有考虑过" |
| 职业发展 | "晋升无望"、"天花板了" | "发展速度一般" | "规划清晰"、"有成长空间" |
| 团队关系 | "和老板不合"、"氛围差" | "有些摩擦" | "团队很好"、"老板支持" |
| 薪酬满意度 | "薪资不匹配"、"被倒挂" | "对薪酬有意见" | "薪酬还可以" |

**战场匹配度评估框架：**
- **能力维度**：核心能力（专业/技术/业务）与当前岗位要求的匹配
- **意愿维度**：对当前岗位的认同感和投入意愿
- **成长维度**：岗位是否提供所需的成长空间

### 3. 职业锚与价值观

#### 3.1 职业锚识别

**8种职业锚类型：**

| 职业锚 | 核心特征 | 识别信号 |
|--------|---------|---------|
| **技术/职能型** | 追求专业精进 | 强调技术深度、不愿做管理 |
| **管理型** | 追求管理责任 | 关注团队规模、喜欢决策 |
| **自主/独立型** | 追求工作自由 | 不喜欢约束、追求灵活性 |
| **安全/稳定型** | 追求稳定可预测 | 关注稳定性、厌恶变化 |
| **创业/创造型** | 追求创造新事物 | 喜欢从0到1、不满足于守成 |
| **服务/奉献型** | 追求帮助他人 | 强调社会价值、关心团队成长 |
| **纯挑战型** | 追求战胜难题 | 喜欢高难度、竞争导向 |
| **生活方式型** | 追求工作生活平衡 | 关注WLB、抵触加班 |

**识别算法：**
```python
def identify_career_anchor(interview_text):
    """
    识别员工的职业锚类型
    """
    anchors = {
        "技术/职能型": 0,
        "管理型": 0,
        "自主/独立型": 0,
        "安全/稳定型": 0,
        "创业/创造型": 0,
        "服务/奉献型": 0,
        "纯挑战型": 0,
        "生活方式型": 0
    }
    
    # 基于关键词匹配
    if "技术深度" in interview_text or "专业" in interview_text:
        anchors["技术/职能型"] += 3
    
    if "团队" in interview_text or "决策" in interview_text:
        anchors["管理型"] += 2
    
    if "灵活" in interview_text or "自由" in interview_text:
        anchors["自主/独立型"] += 3
    
    # ... 其他职业锚的识别逻辑
    
    # 返回得分最高的职业锚
    primary_anchor = max(anchors, key=anchors.get)
    return primary_anchor
```

**输出格式：**
```markdown
#### 职业锚与价值观
- **职业锚类型**：技术/职能型 + 自主/独立型（复合型）
  - 信号：强调技术深度、不愿意做纯管理
  - 信号：喜欢灵活的工作方式，抵触僵化流程
  
- **核心价值观**
  - 🎯 用户导向："从用户场景出发，不是从技术出发"
  - 🚀 创新：喜欢尝试新技术、新方法
  - ⚡ 效率：强调快速迭代，反对流程冗长
  
- **工作动机**
  - 技术成就感：通过技术创新解决问题
  - 用户价值：为用户创造价值是核心驱动力
  - 学习成长：持续学习新技术、新方法
```

### 6. 个人优劣势（v2.6.0 新增）

> **说明**：基于访谈内容事实综合提炼被访谈者的个人优势与不足。

#### 6.1 个人优势

**提取原则：**
- 必须有访谈内容中的事实依据，不可凭空推测
- 至少提取 2 项优势
- 每项优势需包含证据引用

**示例：**
```python
strengths = [
    {
        "name": "AI应用能力",
        "description": "能够熟练运用多种AI工具提升工作效率，并推动团队AI化",
        "evidence": "用户提到：'我现在每天都用CodeBuddy，30分钟能完成以前一周的工作'"
    },
    {
        "name": "产品直觉",
        "description": "对用户需求有敏锐的洞察力，能快速判断产品方向",
        "evidence": "用户强调：'洞察力和判断力很难被AI取代，这是我的核心竞争力'"
    }
]
```

#### 6.2 个人劣势/待提升项

**提取原则：**
- 至少提取 1 项劣势或待提升项
- 如员工未暴露明显劣势，可标注"访谈中未观察到明显不足"
- 区分自我评价与他人评价（如有）

**示例：**
```python
weaknesses = [
    {
        "name": "跨团队影响力",
        "description": "在本团队内影响力强，但跨部门协调能力有待提升",
        "evidence": "用户提到：'跟其他团队配合的时候，推动起来还是比较费劲'"
    }
]
```

### 7. 个人发展诉求（v2.6.0 新增）

> **说明**：从被访谈者的表述中提炼其个人发展意向和期望，包括显性表达和隐性推断。

**提取维度：**

| 维度 | 说明 | 提取信号 |
|------|------|---------|
| **职业发展方向** | 技术专家/管理通道/转岗/创业/其他 | "我想做..."、"未来的方向..." |
| **短期诉求** | 1年内的发展期望 | "最近想..."、"接下来..." |
| **中长期诉求** | 1-3年的发展期望 | "长远来看..."、"3年后..." |
| **晋升意愿** | 强烈/一般/暂无 | "想晋升..."、"职级..." |
| **转岗/活水意向** | 有意向/探索中/无意向 | "想换个方向..."、"对XX部门感兴趣..." |
| **能力提升期望** | 希望提升的能力方向 | "想学..."、"需要提升..." |
| **期望获得的支持** | 来自主管/组织/HRBP的需求 | "希望公司能..."、"如果能给我..." |

**提取规则：**
- 优先提取员工直接表达的发展诉求
- 如员工未直接表达，可基于访谈内容合理推断，但需标注 `[推断]`
- 转岗/活水意向是重要的异动风险信号，需特别关注
- 如果访谈中完全没有涉及发展诉求，标注"访谈中未涉及个人发展诉求"

**示例：**
```python
development_aspirations = {
    "career_direction": "技术专家",
    "career_direction_evidence": "用户说'我还是想在技术上更深入'",
    "short_term": "希望掌握大模型微调能力",
    "short_term_evidence": "用户提到'最近在学大模型相关的东西'",
    "long_term": "成为AI产品技术专家",
    "long_term_evidence": "[推断] 基于其持续关注AI产品化方向",
    "promotion_desire": "一般",
    "transfer_intention": "无意向",
    "skill_improvement": "大模型应用、跨团队影响力",
    "expected_support": "希望有更多参加行业技术会议的机会"
}
```

```
"已生成员工画像，是否需要导出？
  1️⃣ 导出为 Markdown 文件
  2️⃣ 创建为腾讯文档（便于团队协作）
  3️⃣ 补充到员工档案系统
  4️⃣ 暂不需要"

如果选择导出，进一步询问：
"请选择导出目的：
  1️⃣ 补充员工档案
  2️⃣ 团队分析
  3️⃣ 其他"
```

### 2. 提取画像数据

基于访谈内容，使用自然语言处理技术提取各类画像信息。

### 3. 生成画像文档

根据选择的格式（MD/腾讯文档），生成对应的画像文档。

### 4. 导出或同步

根据用户选择，导出文件、创建腾讯文档或同步到HR系统。

---

## 📝 使用建议

1. **画像验证**：在导出前，让员工确认画像的准确性
2. **定期更新**：建议每季度或每半年更新一次员工画像
3. **多源数据**：结合绩效评估、360反馈等多源数据丰富画像
4. **隐私保护**：注意保护员工隐私，仅分享给授权人员
