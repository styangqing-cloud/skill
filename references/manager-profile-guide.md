# 🏢 管理干部画像自动生成指南

本文档说明如何基于访谈内容自动生成**管理干部画像**。管理干部画像适用于 Step 1 识别为 `is_manager = True` 的被访谈者。

> **版本**: v2.7.0 增强版
> **触发条件**: Step 1 管理干部识别结果 `is_manager = True`

---

## 📋 画像结构总览

管理干部画像由六部分组成：

| 部分 | 名称 | 说明 |
|------|------|------|
| **基础身份信息** | 基础身份信息 | 姓名/职位/职级/司龄/部门/管理主体，与普通员工画像统一 |
| **第一部分** | 基础画像（关键指标概览） | 7个核心维度的结构化评估 |
| **第二部分** | 多场景管理表现画像 | 基于访谈事实提取的STAR场景描述 |
| **第三部分** | 个人优劣势（展开版） | 结构化详细描述+证据，与普通员工画像对齐 |
| **第四部分** | 个人发展诉求（展开版） | 7维度结构，与普通员工画像对齐 |
| **第五部分** | 管理风格总结 | 基于多场景表现归纳管理风格类型和管理能力雷达 |
| **第六部分** | 发展潜力与风险 | 综合评估管理干部的发展潜力和潜在风险 |

## 🔍 第一部分：基础画像（关键指标概览）

### 维度说明

| 维度 | 数据来源 | 评估方法 | 输出格式 |
|------|---------|---------|---------|
| **工作投入度** | 时间投入、精力投入、情感投入的相关描述 | 综合分析 | [高/中/低] + 描述 |
| **异动风险** | 不满情绪、外部机会提及、职业发展受阻等信号 | 信号强度综合评估 | [高/中/低] + 描述 |
| **战场** | 业务领域、团队规模、核心职责描述 | 直接提取 | 文字描述 |
| **战场匹配度** | 能力特长与岗位要求的对比分析 | 对比分析 | [高/中/低] + 描述 |
| **战场当前风险与挑战** | 业务/团队面临的关键挑战 | 归纳总结 | 文字描述 |
| **个人优劣势摘要** | 管理者在访谈中展示或提及的优势和不足 | 事实提取 | 关键词概括（详见第三部分） |
| **发展诉求摘要** | 管理者自述或推断的发展期望 | 直接提取+推断 | 一句话概括（详见第四部分） |

### 工作投入度评估

**评估维度：**
```python
def assess_work_engagement(interview_text):
    """
    评估管理者的工作投入度
    """
    dimensions = {
        "时间投入": {
            "high_signals": ["加班", "周末工作", "7x24", "随时响应", "超时"],
            "low_signals": ["准时下班", "不愿加班", "工作生活平衡"],
            "weight": 0.3
        },
        "精力投入": {
            "high_signals": ["全身心", "亲力亲为", "深入一线", "主动关注", "持续优化"],
            "low_signals": ["放手不管", "分心", "精力分散"],
            "weight": 0.35
        },
        "情感投入": {
            "high_signals": ["热爱", "有成就感", "使命感", "自豪", "团队责任"],
            "low_signals": ["倦怠", "疲惫", "没有激情", "例行公事"],
            "weight": 0.35
        }
    }
    return weighted_assessment(dimensions, interview_text)
```

**输出标准：**
- 🟢 **高**：管理者在时间、精力、情感三个维度上展示出高度投入
- 🟡 **中**：部分维度投入度高，部分一般
- 🔴 **低**：多个维度展示出投入度不足

### 异动风险评估

> **异动风险**特指活水转岗或离职的风险

**风险信号分类：**

| 信号类型 | 高风险信号 | 中风险信号 | 低风险信号 |
|---------|-----------|-----------|-----------|
| 不满情绪 | "想换个环境"、"看不到希望" | "有点疲惫"、"不太满意" | "虽然辛苦但值得" |
| 外部机会 | "有人挖我"、"在看机会" | "朋友推荐过"、"不排斥好机会" | "没有考虑过" |
| 职业发展 | "晋升无望"、"天花板了" | "发展速度一般" | "规划清晰"、"有成长空间" |
| 团队关系 | "和老板不合"、"团队氛围差" | "有些摩擦" | "团队很好"、"老板支持" |
| 薪酬满意度 | "薪资不匹配"、"被倒挂" | "对薪酬有意见" | "薪酬还可以" |

**评估算法：**
```python
def assess_transfer_risk(interview_text):
    """
    评估异动风险（活水转岗或离职风险）
    """
    risk_factors = {
        "dissatisfaction": extract_dissatisfaction_signals(interview_text),
        "external_opportunities": extract_opportunity_mentions(interview_text),
        "career_block": extract_career_block_signals(interview_text),
        "team_friction": extract_relationship_signals(interview_text),
        "compensation": extract_compensation_signals(interview_text)
    }
    
    high_count = sum(1 for v in risk_factors.values() if v["level"] == "high")
    
    if high_count >= 2:
        return "高", "存在明确的异动倾向信号"
    elif high_count >= 1 or any(v["level"] == "medium" for v in risk_factors.values()):
        return "中", "存在一定异动风险但未到临界点"
    else:
        return "低", "目前稳定性较好"
```

### 战场匹配度评估

**评估框架：**
- **能力维度**：管理者的核心能力（技术/业务/管理）与当前岗位要求的匹配
- **意愿维度**：管理者对当前岗位的认同感和投入意愿
- **资源维度**：管理者可调动的资源是否支撑岗位目标
- **成长维度**：岗位是否提供管理者所需的成长空间

## 📊 第二部分：多场景管理表现画像

### 场景分类体系

| 场景编号 | 场景名称 | 核心关注点 | 典型识别关键词 |
|---------|---------|-----------|--------------|
| S1 | **业务管理** | 战略制定、目标拆解、业务决策、项目推进 | "业务方向"、"目标拆解"、"项目管理"、"交付"、"OKR" |
| S2 | **团队管理** | 团队建设、招聘培养淘汰、梯队建设、绩效管理 | "团队"、"招人"、"培养"、"绩效"、"淘汰"、"晋升" |
| S3 | **周边协作** | 跨部门协作、向上管理、平级沟通、外部合作 | "跨部门"、"对齐"、"协同"、"老板"、"汇报" |
| S4 | **资源使用** | HC管理、预算管控、工具/平台资源、人力调配 | "HC"、"预算"、"资源"、"人力"、"服务器" |
| S5 | **自我管理** | 时间管理、精力管理、情绪管理、自我成长 | "时间分配"、"压力"、"学习"、"反思"、"平衡" |
| S6 | **其他** | 不属于上述分类的管理场景 | 根据访谈内容灵活识别 |

### STAR 原则描述规范

每个场景的管理表现描述**必须**遵循 STAR 原则：

| 要素 | 说明 | 建议字数 | 占比 |
|------|------|---------|------|
| **S - 情境(Situation)** | 描述事件发生的背景和环境 | 30-80字 | ~15% |
| **T - 任务(Task)** | 描述管理者面临的具体任务或挑战 | 30-60字 | ~12% |
| **A - 行动(Action)** | 描述管理者采取的具体行动和决策过程 | 80-250字 | ~55% |
| **R - 结果(Result)** | 描述行动产生的实际效果和影响 | 40-100字 | ~18% |

**总字数要求**：200-600字，**350字左右最佳**

### 表现倾向标签判定

| 标签 | 符号 | 判定标准 |
|------|------|---------|
| 正向 | 🟢 | 管理行为产生了积极效果，展示出良好的管理能力和判断力 |
| 负向 | 🔴 | 管理行为产生了消极效果或暴露了管理短板和不足 |
| 中立 | 🟡 | 效果不明确，或有好有坏，行为本身不能明确判断为好或差 |

**标签判定算法：**
```python
def determine_sentiment(scene_result: dict) -> str:
    """
    判定场景表现的倾向标签
    
    判定逻辑：
    1. 结果(R)中正面描述 > 负面描述 → 正向
    2. 结果(R)中负面描述 > 正面描述 → 负向
    3. 正面与负面描述相当，或结果不确定 → 中立
    """
    positive_signals = count_positive_outcomes(scene_result["result"])
    negative_signals = count_negative_outcomes(scene_result["result"])
    
    if positive_signals > negative_signals * 1.5:
        return "🟢 正向"
    elif negative_signals > positive_signals * 1.5:
        return "🔴 负向"
    else:
        return "🟡 中立"
```

### 场景提取规则

1. **事实依据原则**：仅提取访谈内容中有明确事实依据的场景，不做无依据的推测
2. **关键原文引用**：每个场景描述必须包含至少一条访谈原文引用作为证据
3. **最少场景要求**：至少提取 2 个场景（如访谈内容不足，注明"访谈内容中此场景信息不足"）
4. **灵活归类**：如某段访谈内容涉及多个场景，优先归入最核心的场景，避免重复描述
5. **S6-其他场景**：仅在前5类无法归类时使用，需明确标注场景性质

## 📝 场景描述详细指南

- 招聘与识人能力

- 绩效管理（含低绩效处理）
- 梯队建设
- 团队文化塑造
- 人才保留

**提取方向示例：**
- "今年有两个核心骨干走了..."

### S3 - 周边协作场景

**关注要点：**

- 预算规划与管控
- 工具/平台资源的争取和利用
- 人力调配与组织效率

**提取方向示例：**
- "今年HC被砍了，我的应对是..."

### S5 - 自我管理场景

**关注要点：**
- 时间管理与精力分配
- 工作压力应对
- 持续学习与自我成长
- 情绪管理
- 工作与生活平衡

- "压力大的时候，我会..."
- "我每天的时间分配大概是..."
- "最近在学习..."

### S6 - 其他场景

**归类标准：**
- 不属于 S1-S5 的管理行为
- 具有独特的管理价值或洞察意义
- 需明确标注场景性质（如"合规管理"、"危机应对"等）

## 💪 第三部分：个人优劣势（展开版，v2.7.0 新增）

> **说明**：基于访谈内容事实综合提炼管理干部的个人优势与不足。与普通员工画像的「个人优劣势」模块结构对齐，采用统一的结构化格式。

### 个人优势

**提取原则：**
- 必须有访谈内容中的事实依据，不可凭空推测
- 至少提取 2 项优势
- 每项优势需包含证据引用

**输出格式：**
```markdown
##### 个人优势
- **[优势1]**：[描述] — 证据："[访谈原文引用]"
- **[优势2]**：[描述] — 证据："[访谈原文引用]"
- **[优势3]**：[描述] — 证据："[访谈原文引用]"
```

### 个人劣势/待提升项

**提取原则：**
- 至少提取 1 项劣势或待提升项
- 如管理者未暴露明显劣势，可标注"访谈中未观察到明显不足"
- 区分自我评价与他人评价（如有）

**输出格式：**
```markdown
##### 个人劣势/待提升项
- **[劣势1]**：[描述] — 证据："[访谈原文引用]"
- **[劣势2]**：[描述] — 证据："[访谈原文引用]"
```

### 与基础画像摘要的关系

- 基础画像中的 **「个人优劣势摘要」** 为关键词式概括（如"优势：技术判断力强、团队凝聚力好 / 劣势：向上管理较弱"）
- 本模块为**展开版**，包含详细描述和访谈原文证据
- 摘要与展开版内容应保持一致

## 🎯 第四部分：个人发展诉求（展开版，v2.7.0 新增）

> **说明**：从被访谈者的表述中提炼其个人发展意向和期望，包括显性表达和隐性推断。与普通员工画像结构对齐，采用 7 维度结构。

### 提取维度

| 维度 | 说明 | 提取信号 |
|------|------|---------|
| **职业发展方向** | 技术管理专家/更高管理层级/业务转型/其他 | "我想做..."、"未来的方向..." |
| **短期诉求** | 1年内的发展期望 | "最近想..."、"接下来..." |
| **中长期诉求** | 1-3年的发展期望 | "长远来看..."、"3年后..." |
| **晋升意愿** | 强烈/一般/暂无 | "想晋升..."、"职级..." |
| **转岗/活水意向** | 有意向/探索中/无意向 | "想换个方向..."、"对XX部门感兴趣..." |
| **能力提升期望** | 希望提升的管理能力方向 | "想学..."、"需要提升..." |
| **期望获得的支持** | 来自上级/组织/HRBP的需求 | "希望公司能..."、"如果能给我..." |

### 提取规则

- 优先提取管理者直接表达的发展诉求
- 如管理者未直接表达，可基于访谈内容合理推断，但需标注 `[推断]`
- 转岗/活水意向是重要的异动风险信号，需特别关注
- 如果访谈中完全没有涉及发展诉求，标注"访谈中未涉及个人发展诉求"

### 与基础画像摘要的关系

- 基础画像中的 **「发展诉求摘要」** 为一句话概括
- 本模块为**展开版**，包含 7 个维度的详细分析和证据

## 🎨 第五部分：管理风格总结（v2.7.0 新增）

> **说明**：基于第二部分多场景管理表现的事实提炼，归纳管理干部的核心管理风格和特征。

### 管理风格标签

**管理风格类型参考（Goleman 六大领导风格）：**

| 风格类型 | 核心特征 | 适用场景 |
|---------|---------|---------|
| **指令型** | 要求立即服从 | 危机时刻、需要快速决策 |
| **愿景型** | 描绘愿景、激发追随 | 变革时期、需要新方向 |
| **亲和型** | 重视人际关系和和谐 | 团队需要修复、提振士气 |
| **民主型** | 通过参与达成共识 | 需要团队认同和创新想法 |
| **标杆型** | 以身作则、设定高标准 | 成员能力强、需要快速结果 |
| **教练型** | 培养他人、长期发展 | 员工需要成长、建立能力 |

> 管理者可能表现出多种风格的组合（如"教练型+愿景型"），应基于多场景表现综合判断。

### 管理能力雷达

基于第二部分的场景表现，对以下 5 个管理维度进行综合评估：

| 管理维度 | 评估标准 |
|---------|---------|
| **业务管理** | 战略制定、目标拆解、业务决策、项目推进能力 |
| **团队管理** | 招聘识人、人才培养、绩效管理、梯队建设能力 |
| **周边协作** | 跨部门沟通、向上管理、平级协作、利益协调能力 |
| **资源使用** | HC管理、预算管控、资源争取、人力调配能力 |
| **自我管理** | 时间管理、精力分配、情绪管理、持续学习能力 |

**评估规则：**
- 仅评估在访谈中有事实依据的场景维度
- 无信息的维度标注"信息不足，暂无法评估"
- 评估等级：🟢 强 / 🟡 中 / 🔴 弱

## 📈 第六部分：发展潜力与风险（v2.7.0 新增）

> **说明**：基于以上所有模块的综合分析，评估管理干部的发展潜力与潜在风险。与普通员工画像的「发展潜力与风险」模块结构对齐。

### 发展潜力

**评估维度：**

| 维度 | 评估内容 | 数据来源 |
|------|---------|---------|
| **管理优势领域** | 管理者在哪些管理维度表现突出 | 第二部分场景表现 + 第五部分管理风格 |
| **成长空间** | 可改进的管理维度和发展方向 | 第三部分劣势 + 第四部分诉求 |
| **晋升可能性** | 综合评估是否具备更高管理层级的潜力 | 综合所有模块 |

### 潜在风险

**风险识别框架：**

| 风险类型 | 关注点 | 来源 |
|---------|--------|------|
| **异动风险** | 活水转岗或离职可能性 | 第一部分基础画像 |

| **团队稳定性风险** | 管理者的管理行为对团队稳定性的影响 | 第二部分场景表现 |

## ⚠️ 注意事项

1. **事实优先**：所有评估必须基于访谈内容中的具体事实，不做主观臆断
2. **平衡呈现**：基础画像中应客观呈现优势和不足，避免单方面美化或贬低
3. **原文支撑**：每个关键判断都需要有访谈原文引用作为支撑
4. **字数控制**：STAR描述严格控制在200-600字范围内，350字左右最佳
5. **隐私保护**：涉及薪酬具体数字、个人隐私等敏感信息，需做脱敏处理
6. **管理干部特殊性**：相比普通员工画像，管理干部画像更关注"管理行为"而非"个人能力"
7. **结构对齐（v2.7.0）**：基础身份信息、个人优劣势、个人发展诉求与普通员工画像保持统一结构化格式
8. **摘要与展开一致性（v2.7.0）**：基础画像中的摘要信息必须与展开版模块内容保持一致
