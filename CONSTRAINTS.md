# Employee Interview Analyzer - 核心约束配置

> **版本**: 2.12.0
> **最后更新**: 2026-03-25

---

## ⭐⭐⭐ 方法论数量强制约束

### 核心规则

**每次访谈分析严格限制最多使用 3 种方法论**

- **主方法论**: 1个（必选）
- **辅助方法论**: 最多2个
- **总计**: 不超过3种方法论
- **⭐ 优先配置**: 1主+1辅=2种（v2.8.0+，仅复杂场景扩展至3种）

### 约束性质

**⚠️ 此为强制约束，任何情况下都不可违背**

- ✅ **自动匹配**: 技能自动匹配方法论时必须遵守
- ✅ **用户指定**: 即使用户要求使用更多方法论，也必须拒绝并遵守限制
- ✅ **场景识别**: 无论访谈场景如何复杂，都不能突破此限制
- ✅ **分析深度**: 宁愿深入应用2-3个方法论，也不浅尝辄止地应用多个

### 约束理由

1. **深度优先**: 深入应用少数方法论比浅层应用多个方法论更有价值
2. **分析聚焦**: 避免分析过于发散，确保洞察聚焦
3. **质量保证**: 方法论过多会导致分析维度分散，影响洞察质量
4. **用户理解**: 3个方法论是用户理解消化能力的合理上限

### 强制执行机制

#### 规则层面

**硬限制常量：**

| 常量 | 值 | 说明 |
|------|------|------|
| `MAX_METHODOLOGIES` | 3 | ⭐⭐⭐ 强制硬限制，不可修改 |
| `PREFERRED_METHODOLOGIES` | 2 | ⭐⭐ v2.8.0 优先配置：1主+1辅 |

**验证规则**：若方法论数量 > `MAX_METHODOLOGIES`，必须拒绝并提示用户选择不超过3种。

#### 用户交互层面

```
当用户要求使用超过3种方法论时：

❌ 错误示例：
用户："用杨三角、韦斯伯德、推拉力、冰山模型、BEM模型分析这份访谈"
分析器："好的，我将使用这5种方法论..." ❌ 错误！

✅ 正确示例：
用户："用杨三角、韦斯伯德、推拉力、冰山模型、BEM模型分析这份访谈"
分析器："抱歉，为确保分析深度和质量，每次分析严格限制最多使用3种方法论。
         我建议从您指定的5个方法论中选择最匹配的3个：
         • 主方法论：杨三角理论（最适合组织诊断）
         • 辅助方法论1：韦斯伯德六盒（系统性组织评估）
         • 辅助方法论2：推拉力理论（如果涉及离职分析）
         
         或者您可以告诉我您最关注的分析维度，我将为您选择最合适的3个方法论组合。" ✅ 正确！
```

#### 自动匹配层面

**匹配规则（决策表）：**

| 步骤 | 规则 | 说明 |
|------|------|------|
| 1 | 基于场景获取主方法论 | 按场景-方法论映射表取值 |
| 2 | 获取候选辅助方法论列表 | 按场景和内容信号筛选 |
| 3 | 评估内容复杂度 | 复杂度 < 阈值 → 最多1辅；≥ 阈值 → 最多2辅 |
| 4 | 按优先级排序并裁剪 | 保留指定数量的辅助方法论 |
| 5 | 组织诊断互斥检查 | M1和M2同时出现时，保留M1移除M2 |
| 6 | 最终验证 | 确保总数 ≤ 3（快速模式下 ≤ 2） |

### 违约后果

**违反此约束将被视为技能实现错误**

- ❌ 分析报告质量下降
- ❌ 用户理解成本增加
- ❌ 洞察聚焦度降低
- ❌ 技能信誉受损

### 例外情况

**⚠️ 无例外情况**

此约束适用于所有场景、所有情况、所有用户请求，没有任何例外。

## 其他核心约束

### 1. 场景自动识别约束

**场景识别必须自动执行，无需询问用户**

- 基于上下文和关键词智能识别访谈场景
- 在 Step 4 分析框架展示时让用户确认或调整
- v2.3.0+ 已去除场景询问步骤，直接自动识别

### 2. 文本修正约束

**必须在分析前自动修正访谈文本**

- 修正错别字
- 修正语句不通
- 记录所有修正内容

### 3. 画像生成约束

**员工画像必须自动生成，无需询问用户**

- 必须包含战场画像（如有信息）
- 必须包含能力、性格、价值观等完整画像
- 组织诊断时必须生成组织画像
- ⭐⭐ **v2.8.0 三轨画像约束**：Step 8 必须根据 Step 1 场景识别结果选择画像策略——离职访谈跳过画像、招聘面试生成候选人综合画像、其他场景沿用双轨画像
- ⭐⭐ **v2.8.0 候选人画像约束**：招聘面试场景必须使用候选人综合画像模板，不得使用员工画像或管理干部画像；岗位匹配分析需基于用户提供的岗位信息
- ⭐⭐ **v2.8.0 方法论优先2种约束**：默认使用1主+1辅=2种方法论，仅复杂场景才扩展至3种
- ⭐ **v2.7.0 结构对齐约束**：普通员工画像和管理干部画像必须统一基础身份信息表格结构
- ⭐ **v2.7.0 关键指标概览约束**：两种画像都必须包含工作投入度、异动风险、战场匹配度、个人优劣势摘要、发展诉求摘要
- ⭐ **v2.7.0 展开版约束**：管理干部画像的个人优劣势和个人发展诉求必须同时生成摘要（基础画像中）和展开版（独立模块），内容保持一致
- ⭐⭐⭐ **v2.10.0 强制执行约束**：除离职访谈外，所有场景的 Step 8 画像生成为强制步骤，不可跳过；包括AI使用情况等特殊场景也必须生成画像
- ⭐ **v2.10.0 画像重试约束**：画像生成失败时，必须提供重试机制和错误提示，不可静默跳过

### 4. 管理干部画像HR系统对齐约束 ⭐⭐⭐（v2.11.0 新增）

**管理干部画像的多场景管理表现必须对齐HR系统维度**

- ⭐⭐⭐ **行为标签三分类**：每条行为记录的标签必须使用HR系统三分类——`正向行为` / `需关注和待提升行为` / `无标签`，不得使用旧版的🟢正向/🔴负向/🟡中立标记
- ⭐⭐⭐ **场景类型可多选**：每条行为记录的场景类型改为可多选字段——`业务管理` / `团队管理` / `周边协作` / `资源使用` / `自我管理` / `其他`，替代原有的单场景分类（S1-S6）
- ⭐⭐ **STAR原则描述**：行为描述必须严格按 STAR 原则（Situation情境 → Task任务 → Action行动 → Result结果/影响）进行结构化描述
- ⭐⭐ **发生时间字段**：每条行为记录必须包含「发生时间」字段，从访谈中提取或推断行为发生的大致时间
- ⭐ **独立编号**：每条行为记录必须独立编号，支持同一访谈提取多条行为记录
- ⭐ **标准化格式**：行为记录采用表格+描述的标准化格式，便于HR系统录入
- ⭐ **场景名称对齐**：管理场景名称去掉"场景"后缀（如"业务管理场景"→"业务管理"），与HR系统字段名完全一致

### 5. 可选步骤批量询问约束 ⭐⭐（v2.12.0 合并）

**Step 10/11/13 合并为一次批量询问**

- ⭐⭐ **v2.12.0 批量询问**：Step 9 完成后，将数据可视化（Step 10）、结论文章（Step 11）、腾讯文档（Step 13）合并为一次批量决策询问
- 用户通过编号选择需要的步骤（如"1,3"），未选中的默认跳过
- 快速模式下不发起询问，直接全部跳过
- 数据可视化基于分析结果智能判断图表类型（雷达图、柱状图、折线图等）
- 结论文章字数800-1500字，必须包含5部分（访谈背景、核心发现、深度分析、建议与展望、总结）
- 腾讯文档使用 `tencent-docs` skill 创建，标题格式：`{访谈对象}-访谈分析报告-{YYYY-MM-DD}`

### 6. 流程执行约束 ⭐⭐（v2.10.0 新增）

**分析流程的每个步骤必须按顺序执行，不可遗漏**

- ⭐⭐ **流程状态追踪**：必须维护流程状态变量，记录每个步骤的执行状态
- ⭐⭐ **进度提示**：在关键节点输出当前进度（Step X/13），确保用户了解分析进展
- ⭐ **错误处理**：步骤执行失败时必须提供明确错误提示和恢复建议
- ⭐ **断点续行**：支持用户中断后继续执行，从上一个已完成步骤之后恢复
- ⭐⭐ **询问状态追踪**：Step 10/11/13 的询问结果必须记录，确保每个询问都被正确处理

### 7. 快速分析模式约束 ⭐⭐（v2.12.0 新增）

**快速模式的触发和行为受以下约束控制**

- ⭐⭐⭐ **触发条件**：复杂度评估得分 ≤ 3（自动触发），或用户明确请求（`"快速分析"` `"简单分析"`）
- ⭐⭐⭐ **场景限制**：组织诊断和综合分析场景 **禁止进入** 快速模式（即使用户手动请求也必须拒绝）
- ⭐⭐⭐ **置信度限制**：场景识别置信度 < 0.7 时 **禁止进入** 快速模式（需用户确认场景，走标准模式）
- ⭐⭐ **方法论限制**：快速模式下强制使用 1主+1辅=2种方法论，不可扩展至3种
- ⭐⭐ **跳过策略**：
  - Step 4（框架确认）→ 跳过确认，自动采用推荐框架
  - Step 6（交叉验证）→ 简化执行，仅做关键矛盾检测
  - Step 9/10/11/13 → 默认跳过，不发起询问
- ⭐⭐ **不可跳过步骤**：Step 1/2/3/5/7/8/12 在快速模式下 **不可省略**
- ⭐ **模式切换**：用户可随时通过 `"详细分析"` `"完整模式"` 切换回标准模式
- ⭐ **标注义务**：快速模式下报告开头必须附注快速模式标记和复杂度评分
- ⭐ **检查清单简化**：Step 12 输出前仅检查 Step 1/2/3/5/7/8 的执行状态，跳过项不纳入

## 合规检查清单

在生成分析报告前，必须检查：

### 方法论约束
- [ ] 方法论总数是否 ≤ 3（快速模式下 ≤ 2）
- [ ] 主方法论是否为1个
- [ ] 辅助方法论是否 ≤ 2个（快速模式下 ≤ 1个）
- [ ] 是否已向用户说明此约束（如果用户指定超过3种）
- [ ] 是否已拒绝用户不合理的方法论数量要求

### 画像约束
- [ ] ⭐ Step 8 画像生成：非离职场景是否已强制执行
- [ ] ⭐ 画像策略是否与 Step 1 场景识别结果一致（三轨判定）
- [ ] ⭐ 管理干部画像行为标签是否使用HR系统三分类（v2.11.0）
- [ ] ⭐ 管理干部画像场景类型是否支持多选（v2.11.0）
- [ ] ⭐ 行为描述是否按STAR原则（v2.11.0）
- [ ] ⭐ 行为记录是否包含发生时间字段（v2.11.0）

### 流程约束（标准模式）
- [ ] ⭐ Step 10/11/13 批量询问：已在 Step 9 完成后向用户提供可选步骤菜单
- [ ] ⭐ 用户选择的步骤已执行，未选中的已跳过
- [ ] ⭐ Step 12 完整报告：已输出
- [ ] ⭐ 流程进度是否已在关键节点输出（Step X/13）

### 快速模式约束（v2.12.0）
- [ ] ⭐ 复杂度评估是否在 Step 1 完成后执行
- [ ] ⭐ 组织诊断/综合分析场景是否已阻止进入快速模式
- [ ] ⭐ 场景识别置信度 < 0.7 时是否已阻止进入快速模式
- [ ] ⭐ 方法论数量是否限制为2种
- [ ] ⭐ 报告开头是否附注快速模式标记和复杂度评分
- [ ] ⭐ Step 1/2/3/5/7/8/12 是否均已执行
